# ClawMeeting 使用引导

<description>
ClawMeeting 智能会议协商插件的使用引导。当用户首次使用 ClawMeeting、询问如何使用会议功能、或说"帮我约会议"但尚未绑定邮箱时激活本技能。触发词：ClawMeeting、会议助手、约会议、绑定邮箱、会议协商。
</description>

ClawMeeting 是用户的智能会议助手，能帮用户：
- 一句话约会议，自动收集所有人的空闲时间
- 自动处理邀请（查日程、回复空闲时间）
- 智能协商时间冲突，需要用户拿主意时才通知
- 隐私优先——服务端只看到可用时间段，看不到忙什么

## 引导流程

当用户首次接触 ClawMeeting 或不知道怎么用时，按以下顺序引导：

### 1. 检查绑定状态

先调用 `check_and_respond_tasks`（无参数），如果返回"尚未完成身份绑定"，说明用户还没绑定邮箱。

### 2. 引导绑定（两步验证码）

告诉用户需要绑定邮箱才能开始使用，请用户提供邮箱地址。

用户提供邮箱后：
1. 调用 `bind_identity`（传入邮箱）→ 发送验证码到用户邮箱
2. 请用户查收邮件，告知验证码
3. 调用 `verify_email_code`（传入邮箱 + 验证码）→ 完成绑定

绑定成功后，插件会在后台自动检查新的会议邀请。

### 3. 告知功能

绑定成功后，告诉用户可以做什么：
- **发起会议**："帮我约 bob@example.com 明天开个会"
- **查看邀请**："有没有新的会议邀请？"
- **后台自动处理**：别人约你开会时自动回复空闲时间，需要你决策时才通知你

## 用法示例

| 用户说 | 效果 |
|--------|------|
| "帮我绑定邮箱 xxx@xxx.com" | 发送验证码，开始绑定流程 |
| "帮我约 Bob 和 Charlie 明天开会" | 发起会议协商 |
| "帮我约个 30 分钟的架构讨论会，尽量下午" | 发起会议，附带时间偏好 |
| "有没有新的会议邀请？" | 手动检查待办任务 |
| "可以，就这个时间" | 接受协调方的时间建议 |
| "周四不行，我只有周五上午有空" | 提出新的时间方案 |
| "这个会议我不参加了" | 拒绝参加 |

## 常见问题

**后台轮询会消耗 Token 吗？**
不会。后台检查是纯 HTTP 请求，不经过大模型。

**别人约我开会，需要手动回复吗？**
大部分情况不需要。插件自动读取日程并上报空闲时间。只有时间冲突需要用户取舍时才通知。

**日程隐私安全吗？**
安全。插件只向服务端发送"可以开会的时间段"，不发送忙什么、会议标题等隐私信息。

## 注意事项

- 后台轮询不消耗 Token（纯 HTTP）
- 用户的日程隐私不会泄露（只发送可用时间段）
- 每个会议用标题称呼，不需要用户记 ID
