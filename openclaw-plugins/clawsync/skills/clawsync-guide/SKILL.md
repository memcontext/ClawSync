# ClawSync 使用引导

<description>
ClawSync 智能会议协商插件的使用引导。当用户首次使用 ClawSync、询问如何使用会议功能、或说"帮我约会议"但尚未绑定邮箱时激活本技能。触发词：ClawSync、会议助手、约会议、绑定邮箱、会议协商。
</description>

## 引导流程

当用户首次接触 ClawSync 或不知道怎么用时，按以下顺序引导：

### 1. 检查绑定状态

先调用 `check_and_respond_tasks`（无参数），如果返回"尚未完成身份绑定"，说明用户还没绑定邮箱。

### 2. 引导绑定

告诉用户：
> ClawSync 是你的智能会议助手，可以帮你一句话约会议、自动处理邀请。
> 要开始使用，请告诉我你的邮箱地址，我来帮你注册。

用户提供邮箱后，调用 `bind_identity` 完成绑定。

### 3. 告知功能

绑定成功后，告诉用户可以做什么：
- **发起会议**："帮我约 bob@example.com 明天开个会"
- **查看邀请**："有没有新的会议邀请？"
- **后台自动处理**：别人约你开会时自动回复空闲时间，需要你决策时才通知你

### 4. 常见用法示例

| 用户说 | 效果 |
|--------|------|
| "帮我绑定邮箱 xxx@xxx.com" | 注册身份 |
| "帮我约 Bob 和 Charlie 明天开会" | 发起会议协商 |
| "帮我约个 30 分钟的架构讨论会，尽量下午" | 发起会议 + 时间偏好 |
| "有没有新的会议邀请？" | 手动检查待办 |
| "可以，就这个时间" | 接受建议 |
| "周四不行，我只有周五上午有空" | 提出新方案 |

## 注意事项

- 后台轮询不消耗 Token（纯 HTTP）
- 用户的日程隐私不会泄露（只发送可用时间段）
- 每个会议用标题称呼，不需要用户记 ID
